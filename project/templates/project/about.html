Technologies Used (About)

In this website we used a few technologies, including, but not limited to, HTML, CSS, Django and JavaScript.

We used CSS and HTML to format the pages to the way they are now, for that we created a single page, using HTML, and then used it as a basis to make more pages using Django.

Django was also used to connect the pages of the website, to read out the data bases, to add onto the data bases, to define how the pages were rendered, etc... Django was the most complicated, but also most important, part of this website as it servers as the basis for the whole project.

JavaScript, unfortunately, wasnt used to its potential. We planned to create the incrementation of the quiz using Javascript and the SPA to see the lists, but due to time constraints, we werent able to do those things.

(Updated Technologies Used)
Technology Usage
HTML Used to style the pages in the format we need them to be. Also used in conjunction with Django in order to reduce the amount of information repeated between pages using a single page as the basis.
CSS Used to format the elements in the HTML pages by creating classes, ids and other certain identifying elements so that we can format each individual section to our liking.
Django Framework of the website, being used to store the HTML pages, the images, the CSS and the javascript used in the whole website. Also used to make sure the pages are viewed correctly and that the information that is needed can be carried between them.
Javascript Not used to its potential as we only planned to use it to increment the answers of the "Quiz" list.

Techniques Used (About)

To make this website, we depended mostly on the use of the CSS property "position: absolute", especially for the Hub page, and the CSS Grid and Flexbox layout, for the rest of the pages.

To make the Hub, we had to use the position absolute to put the planets, and their names, in the position we wanted them to be in. To make it so that they were in the same position no matter what screen we were in, we used the "%" values to change their "top" and "left" values, which are the properties we used to put the planets in their place.

To make the pages, we had to focus on the CSS Grid, and how we would move them depending on if they´re on mobile or if they´re on laptop. For that we just had to define the different sections of the page, including the "title", which represents the title of the page, and the "header", which are the images to the left.

To make the forms, aka the data bases for the pages, we had to focus on the movement between the pages, and how that data would be seen by the user. For that, we always show the list of the data base first, followed by the option to be able to add onto that list using a form. Addind onto the list was done using "Django".

(Updated Techniques Used)
Caracteristics Elements/Tecniques Examples
Layouts We used CSS Grid and Flexbox to make the layout of all pages except the Hub. This page is an example, as we made the title and the main as parts of the full layout.
Mobile Accessibility We also used CSS Grid and Flexbox, along with media querys, to ensure that the website would run well with mobile users. If we put these pages into a mobile phone, they will appear in a different way.
Selectors All CSS selectors were used during this website, including, but not restricted to, the "class" selector, the "id" selector and the "pseudo-class" selector. An example, taken directly from the CSS code, could be the id for the sun, that being -> "#sun".
Selector Combinations Most CSS selector combinations were used during this website, including, but not restricted to, the "adjacent" combination ("~") and the "descendent" combination (" "). The only one not used was the "immediatly-adjacent" combination ("+"). An example, taken directly from the CSS code, could be "main section form fieldset p ~ p".
Tags Most of the HTML 5 tags, except the "Article" tag, were used in the HTML code. For an example, all the main content in this page was written inside the "main" tag.
Django Used the views and models off of "views.py" and "models.py" such as "User" representing each contact via its variables with the type "models.CharField()". The model/data base used to hold the contacts inserted has the variable -> userName = models.CharField("User Name", max_length=40).
Javascript Wasnt used to increment the amount of answers on the list and then show it on the site. Not used in the website.
Animations We created two animations, one, that being the rotate animation, just made the image rotate, while the other, that being the border animation, just made the border colors change when they were hovered over. The rotate animation was used for some of the planets in the Hub, while the border animation was strictly used for the image gallery, when the images are hovered over.
